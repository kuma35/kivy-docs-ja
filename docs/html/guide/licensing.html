Package licensing¶

警告

これは法的に権威のあるガイドではありません! Kivyの組織と著者と貢献者は、 ここに記載されている知識または情報またはアドバイスの欠如について一切責任を負いません。 このガイドは単なる参考情報であり、 経験の浅いユーザーを保護することを目的としています。

あなたのコードだけなら、 含まれている他のソフトウェアのライセンス情報や著作権表示を含める必要はないかもしれませんが、 バイナリになると別です。 バイナリ (.exe、.app、.apk、...) が作成されると、 Kivyやその依存関係、およびアプリケーションが使用するその他のパッケージが含まれます。

それらの一部は、 あなたアプリのどこかに著作権表示を含める必要がある方法でライセンスされています(またはそれ以上の事が必要です)。 バイナリを配布する前に、 あなたのソースコードでない全ての作成済みファイル (.dll, .pyd, .so, ...) を確認し、ファイルが属するライセンスにおいて必要な場合、適切な著作権表示を含めてください。 このようにすることで、 あなたは Kivy deps のライセンス要件を満たすことができます。

Dependencies¶

すべての依存関係は、 Kivy がサポートする各プラットフォームにて、少なくともその一部が使用されます。 したがって、 ライセンスに準拠する必要があります。 ほとんどの場合、 あなたのアプリに著作権表示を貼り付けるだけで、 コードを書いたふりをする必要はありません。

  • docutils

  • `pygments https://github.com/pygments/pygments/blob/master/LICENSE`_

  • sdl2

  • glew

  • gstreamer (if used)

  • image & audio libraries(e.g. SDL_mixer has them)

あなたは、 おそらく画像(image)ライブラリと音声(audio)ライブラリを手動でチェックする必要があるでしょう(ほとんどのライブラリは lib で始まります)。 それらに属する LICENSE* ファイルは PyInstaller に含まれているはずですが、 python-for-android には含まれていないため、 あなたはそれらを見つける必要があります。

Windows (PyInstaller)¶

いくつかの Windows API 機能にアクセスするために、 Kivy は pypiwin32 パッケージを使用します。 このパッケージは、 PSF license の下でリリースされています。

Visual Studio Redistributables¶

Visual Studio (official) でコンパイルされた Python には、 Microsoft のファイルが含まれており、 CRTlicense に記載されている特定の条件下でのみ再配布することが許可されています。 ファイルの名前と Py2 CRT license または Py3 CRT license (使用するインタープリターに応じて) の言い換えバージョンを含め、 アプリケーションのエンドユーザーの要件を満たすためにこれらを提示する必要があります。

Other libraries¶

注釈

「あなたは直接使用しない」が、 パッケージ化後に存在する、 たとえば Windows 上の PyInstalle などの、 他のライブラリの属性を追加してください。

Linux¶

Linux には多くのディストリビューションがあるため、 すべてのディストリビューションに対応する正確なガイドはありません。 これは RPi にも当てはまります。 ただし、 パッケージの作成方法 (PyInstaller 使用も含む) に応じて、 バイナリを含めるか含めないかの 2 つの方法に単純化されます。

バイナリが含まれている場合は、 ソースではないすべてのファイル (.so など) を確認し、 それが属するライセンスを見つける必要があります。 ライセンスにより、 そのライセンスの要件に応じて、 おそらくあなたのアプリケーションにその帰属を表示する必要があります。

バイナリが含まれていない場合 (たとえばあなたは .deb パッケージとしてアプリをパッケージ化できます)、 situation bad for your user があります。 他のライセンスの条件を満たしているかどうか、 たとえばアプリに著作権の帰属を含めるかどうかは、 あなた次第です。

Android¶

APK または AAB は単なるファイルのアーカイブであるため、 それらからファイルを抽出し、 (Windows の再配布可能ファイルのように) すべてのファイルを確認できます。

private.tar は、 インクルードされているすべてのファイルを含んでいます。 それらのほとんどは、 KivyまたはPythonまたはあなたのソース・コードに関連していますが、 確認する必要がないものです。

apk: APK/assets/private.tar

aab: AAB/base/assets/private.tar

APK/lib/* または AAB/base/lib/* にある、 Kivy によって直接または SDL2 を介して含まれる他のライブラリが含まれています。 そのほとんどは依存関係に関連しているか、 python-for-androidによって生成され、 そのソース（とライセンス）の一部となっています。

警告

libpybundle.so は、実際には Python の modules と site-packages を含む tarball です。 あなたは、 おそらくライセンス目的で tar -xvf libpybundle.so を介して検査したいと思うでしょう。

Avoiding binaries¶

サードパーティのものを含むディストリビューションを完全に作成しないようにすることで、 このライセンス処理を回避する方法があるかもしれません。 Python を使用すると、 必要な依存関係のみをリストする __main__.py + setup.py を使用したコードのみであるモジュールを作成できます。

このようにすると、 あなたのアプリ (あなたのコード) を依然として配布でき、 他のライセンスを気にする必要がない場合があります。 あなたのコードと依存関係の組み合わせは、「配布」(distribution)ではなく「利用」(usage)として指定できます。 ただし、 ほとんどの場合、 モジュールを実行するために実行環境を構築するユーザーが、 ライセンスを満たす責任を負う事になります。 あなたがユーザーのことを気にかけているなら、 少しペースを落として consequences についてもっと読んだ方がいいかもしれません。
